# ETF 两阶段优选研究框架 - 项目说明书

## 项目概述

**项目名称**：传统机器学习与大语言模型融合的 ETF 优选研究框架
**项目性质**：学术研究型毕业设计项目
**核心目标**：在传统量化因子模型预测能力有限的前提下，引入大语言模型进行二次排序，检验是否能够提升投资组合的回测表现与风险收益特征。

本项目围绕 ETF 横截面优选问题展开，采用“传统机器学习预测 + 大语言模型二次排序”的两阶段研究思路。项目首先利用 ETF 日线行情数据构建技术因子与未来收益标签，并基于第一阶段传统模型完成多目标预测；随后在回测阶段，将传统模型筛选出的候选 ETF 交由大语言模型进行横截面比较与重排序，进而对比仅传统模型策略与两阶段策略在收益和风险指标上的差异。项目同时实现了 LLM 排序日志记录、解释内容结构化存储及 HTML 报告生成功能，用于增强实验过程的可追踪性与可解释性。

当前仓库中的第一阶段传统模型基线为 `random_forest`。项目在工程结构上已经为后续扩展其他常见传统模型保留统一的命名、路径与参数入口，后续扩展方向包括 `linear`、`lightgbm`、`xgboost`。

## 核心功能模块

### 1. 数据准备模块

* **数据源**：akshare 提供的 ETF 日线行情数据接口
* **ETF样本**：100 只主流 ETF，覆盖宽基指数、行业主题、跨境及策略类品种
* **数据范围**：当前仓库内因子总表覆盖时间区间为 2023-02-22 至 2026-02-10
* **因子计算**：基于滚动窗口构造动量、波动率、成交量、均线、换手率等技术指标
* **标签构造**：支持次日收益、未来 5 日收益、未来 10 日收益及未来波动率变化预测目标
* **模拟数据支持**：`scripts/dev_tools/generate_mock_data.py` 可生成独立测试用样本

### 2. 传统机器学习模块

* **模型选择**：当前第一阶段基线模型为随机森林回归器（RandomForestRegressor）
* **扩展方向**：后续可在统一目录与参数框架下扩展 `linear`、`lightgbm`、`xgboost`
* **预测目标**：同时处理 5 个目标变量

  * `Y_next_day_return`
  * `Y_future_5d_return`
  * `Y_future_10d_return`
  * `Y_future_5d_vol_change`
  * `Y_future_10d_vol_change`
* **训练方式**：按日期进行 80/20 时间切分，避免未来信息泄漏
* **评估指标**：R²、MAE、RMSE 及特征重要性分析
* **输出内容**：预测结果文件、训练/测试集划分信息、特征重要性图与预测对比图
* **路径管理**：通过 `traditional_model_config.py` 统一维护默认模型、目录路径和命名约定

### 3. 大语言模型排序模块

* **API服务**：DeepSeek Chat API
* **Prompt设计**：将模型定位为“ETF 横截面比较与排序器”，强调仅依据输入因子进行判断
* **约束条件**：

  * 禁止调用外部知识和联网搜索
  * 禁止进行时间序列外推
  * 输出合法 JSON 排序结果
* **日志记录**：保存 Prompt、原始响应和解析结果
* **模拟模式**：支持本地 mock 排序逻辑，用于无 API 条件下的流程验证

### 4. 两阶段策略框架

* **第一阶段**：传统模型按预测值排序，筛选候选 ETF
* **第二阶段**：对候选 ETF 进行 LLM 重排序，形成最终组合
* **对比基准**：仅使用传统模型预测值排序的策略 A
* **异常处理**：在线 LLM 调用失败时自动回退至第一阶段排序结果
* **参数控制**：支持传统模型名称、目标变量、调仓频率、预选数量、最终组合数量、是否使用 LLM、是否启用模拟排序等配置

### 5. 回测评估系统

* **调仓频率**：支持日频、周频、月频，用于控制组合更新节奏与回测采样间隔。
* **持有期**：默认 5 个交易日，与默认收益预测目标 `Y_future_5d_return` 保持一致。
* **组合权重**：采用等权重配置，即组合内各 ETF 按相同权重持有。
* **绩效指标**：

  * **年化收益率**：衡量策略在全年维度上的收益水平，用于反映组合的整体盈利能力。
  * **年化波动率**：衡量策略收益波动程度，用于反映组合风险水平。
  * **夏普比率**：衡量单位风险对应的超额收益水平，用于比较不同策略的风险收益效率。
  * **最大回撤**：衡量策略从历史净值高点回落到低点的最大跌幅，用于反映极端亏损风险。
  * **胜率**：衡量各调仓期中取得正收益的比例，用于反映策略在单期层面的稳定性。
* **结果展示**：支持累计收益曲线、收益分布图和绩效对比图，分别用于展示策略净值演化、单期收益分布特征以及不同策略之间的核心指标差异。

### 6. LLM 可解释性模块

* **功能定位**：在排序结果基础上生成展示型解释内容，增强研究过程透明度
* **成本控制**：解释功能默认关闭，通过命令行参数显式开启
* **解释输出**：

  * 完整 JSON 响应
  * 文本摘要
  * ETF 解释明细表
  * 结构化解释文件
  * 单日报告与汇总 HTML 报告
* **配置管理**：`explanation_config.py` 当前主要作为解释功能的辅助配置与成本估计工具存在，尚未作为主流程命令行执行时的实际配置入口接入
* **验证支持**：提供 `simple_validation.py` 与 `test_explanation_feature.py` 进行功能检查

## 实验设计

### 研究假设

1. 传统量化因子模型对 ETF 未来收益的预测能力存在局限，收益类目标的 R² 可能接近 0 或为负。
2. 大语言模型具备基于多因子截面数据进行综合比较与排序的能力。
3. 传统模型与 LLM 组合构成的两阶段框架有望改善风险调整后收益表现。

### 对比策略

* **策略 A（基准）**：仅传统机器学习模型排序，直接选取最终组合
* **策略 B（实验）**：传统模型预筛选 + LLM 二次排序，再选取最终组合
* **策略 B-Mock（验证）**：传统模型预筛选 + 本地 mock 排序，用于离线测试运行链路

### 回测参数

| 参数      | 默认值                  | 说明                     | 可配置选项                                                            |
| ------- | -------------------- | ---------------------- | ---------------------------------------------------------------- |
| 传统模型    | `random_forest`      | 第一阶段传统模型名称             | 后续可扩展 `linear`, `lightgbm`, `xgboost`                            |
| 预测目标    | `Y_future_5d_return` | 未来 5 日收益率              | `Y_next_day_return`, `Y_future_5d_return`, `Y_future_10d_return` |
| 调仓频率    | `W`                  | 周频调仓                   | `D`, `W`, `M`                                                    |
| 持有期     | 5 个交易日               | 当前代码中固定为 5 个交易日，与默认收益目标保持一致 | 如需调整，需修改代码中的对应常量                                          |
| 第一阶段选取  | 50 只                 | 传统模型预选数量               | 整数参数                                                             |
| 最终组合    | 10 只                 | 最终持有数量                 | 整数参数                                                             |
| ETF样本规模 | 100 只                | 样本池规模                  | 可通过修改 `etf_list.csv` 扩展                                          |
| LLM使用   | 开启                   | 是否执行在线排序               | 可关闭                                                              |
| Mock模式  | 关闭                   | 是否使用本地模拟排序             | 可开启                                                              |
| 解释功能    | 关闭                   | 是否生成解释与报告              | 可开启                                                              |
| 解释日期    | 默认抽样                 | 指定或抽样生成解释的调仓日          | `--explanation-date` / `--explanation-sample-size`               |
| LLM超时   | 自动估算                 | DeepSeek API 请求超时时间（秒） | 正整数；也可留空使用自动估算                                                   |
| 权重分配    | 等权重                  | 组合内 ETF 权重             | 当前仅支持等权                                                          |
| 交易成本    | 暂不考虑                 | 手续费与冲击成本               | 当前未建模                                                            |

## 使用方法

### 环境设置

```bash
# 1. 克隆项目
git clone git@github.com:MogekoQwQ/etf.git
cd etf-main

# 2. 创建虚拟环境
py -3.10 -m venv .venv

# 3. 激活虚拟环境
# PowerShell:
.\.venv\Scripts\Activate.ps1

# 4. 安装依赖
pip install -r requirements.txt

# 5. 进行环境诊断
python scripts/dev_tools/check_env.py

# 6. 设置 DeepSeek API 密钥（规范用法，推荐在在线 LLM 排序时显式设置）
# PowerShell:
$env:DEEPSEEK_API_KEY="your_api_key_here"
```

### 快速开始

```bash
# 全流程执行
python scripts/run_pipeline.py

# 跳过数据下载（当原始数据已存在时）
python scripts/run_pipeline.py --skip-download

# 跳过 LLM，仅执行传统模型回测链路
python scripts/run_pipeline.py --skip-download --skip-factors --skip-traditional --skip-llm

# 使用本地 mock 排序验证两阶段流程
python scripts/run_pipeline.py --skip-download --skip-factors --skip-traditional --mock-llm

# 显式指定第一阶段传统模型
python scripts/run_pipeline.py --traditional-model random_forest

# 启用解释功能并对指定日期生成解释
python scripts/run_pipeline.py --enable-explanations --explanation-date 2025-12-05

# 启用解释功能并抽样生成少量解释
python scripts/run_pipeline.py --enable-explanations --explanation-sample-size 3

# 启用解释功能并显式设置 LLM 超时（秒）
python scripts/run_pipeline.py --enable-explanations --llm-timeout 180
```

### 分步执行

```bash
# 0. 环境诊断
python scripts/dev_tools/check_env.py

# 1. 数据下载（需要网络与 akshare 可用）

python scripts/compute_etf_factors.py

python scripts/two_stage_backtest.py --traditional-model random_forest

python scripts/two_stage_backtest.py --traditional-model random_forest --no-llm

# 6. 两阶段回测对比（mock 模式）
python scripts/two_stage_backtest.py --traditional-model random_forest --mock-llm

# 7. 启用解释功能的两阶段回测
python scripts/two_stage_backtest.py --traditional-model random_forest --enable-explanations

python scripts/dev_tools/test_api.py

# 9. 解释功能验证
python scripts/dev_tools/simple_validation.py
python scripts/dev_tools/test_explanation_feature.py

# 10. 生成解释 HTML 报告
# 这两个报告脚本的默认相对路径按“当前工作目录在 scripts/ 下”设计。

# 11. 生成两阶段回测 HTML 汇总报告
python scripts/two_stage_reporter.py --input-dir results/traditional_models/random_forest/two_stage --output results/traditional_models/random_forest/two_stage/report_summary.html

# 12. 生成模拟数据
python scripts/dev_tools/generate_mock_data.py
```

## 核心算法与实现

### 因子计算算法

```python
# 主要技术指标因子
1. 动量因子：momentum_20 = 收盘价在20日窗口下的涨跌幅
2. 波动率因子：volatility_20 = 20日收益率滚动标准差
3. 成交量因子：volume_mean_20 = 20日成交量均值
4. 收益均值因子：return_mean_20 = 20日涨跌幅均值
5. 振幅均值因子：amplitude_mean_20 = 20日振幅均值
6. 换手率均值因子：turnover_mean_20 = 20日换手率均值
7. 趋势因子：MA_5, MA_10 = 5日与10日移动平均线
8. 标签变量：未来1日、5日、10日收益率与未来波动率变化
```

### 两阶段策略算法

```python
# 策略A：仅传统模型
def execute_strategy_a(df, date):
    1. 筛选调仓日当日样本
    2. 按预测值降序排序
    3. 选取前 TOP_N_FINAL 只 ETF

# 策略B：两阶段排序
def execute_strategy_b(df, date):
    1. 第一阶段：按预测值排序，选取前 TOP_N_FIRST 只 ETF
    2. 第二阶段：调用 LLM 或 mock 排序模块进行重排
    3. 选取重排后的前 TOP_N_FINAL 只 ETF
    4. 若在线调用失败，则回退至第一阶段排序结果
```

## 限制与注意事项

### 数据限制

1. **样本规模**：当前样本池为 100 只 ETF，覆盖面较广，但仍不能代表全部市场品种。
2. **时间跨度**：当前数据覆盖约 3 年，无法囊括所有市场状态。
3. **因子设计**：当前以技术指标因子为主，尚未纳入宏观、基本面或事件类因子。
4. **数据获取**：原始数据下载依赖 akshare 及网络环境稳定性。
5. **数据质量**：金融数据可能存在缺失值、异常值与停牌影响，需在研究中关注。

### 研究假设

1. **LLM能力假设**：假设大语言模型能够对多因子截面特征进行有效综合比较。
2. **市场改进空间假设**：假设传统因子模型存在可由二阶段排序改善的空间。
3. **实验性质假设**：本项目以研究框架验证为主，不直接等同于实盘交易系统。
4. **可解释性假设**：假设结构化解释与报告输出有助于增强研究结论的透明度。

## 故障排除

### 常见问题

1. **导入错误**：请确认已在虚拟环境中执行 `pip install -r requirements.txt`。
2. **环境异常**：可执行 `python scripts/dev_tools/check_env.py` 检查 Python 版本、关键依赖与虚拟环境状态。
3. **预测文件缺失**：若 `data/predictions/random_forest/test_set_with_predictions.csv` 不存在，应先运行 `python scripts/train_traditional_multi_target_backtest.py --traditional-model random_forest`。
4. **API调用失败**：规范用法下，在线模式应显式设置 `DEEPSEEK_API_KEY` 并保证网络连通性；可使用 `--mock-llm` 进行离线验证。若主要报错为读取超时，可显式增加 `--llm-timeout`（例如 `--llm-timeout 180`）。
5. **解释数据缺失**：生成 HTML 报告前，应确保 `results/traditional_models/random_forest/two_stage/llm_logs/explanations/` 下存在对应日期的数据目录。
6. **路径错误**：若出现数据文件不存在的提示，应确认是否从仓库根目录按 `python scripts/...` 方式执行；对于 `explanation_reporter.py` 与 `two_stage_reporter.py`，若直接从仓库根目录运行，建议显式传入路径参数。

### 调试建议

1. **先执行环境诊断**：使用 `python scripts/dev_tools/check_env.py` 验证当前 Python 与依赖状态。
2. **先执行轻量验证**：使用 `python scripts/dev_tools/simple_validation.py` 与 `python scripts/dev_tools/test_explanation_feature.py` 检查解释模块。
3. **离线验证运行链路**：优先使用 `--no-llm` 或 `--mock-llm` 验证整体流程。
4. **检查日志文件**：若在线排序结果异常，可查看 `results/traditional_models/random_forest/two_stage/llm_logs/` 下的 Prompt 与响应文件。
5. **超时优先排查**：解释功能开启后 Prompt 和输出都会变大，若出现读取超时，可先提高 `--llm-timeout`，不传时程序会按请求规模自动估算超时。
6. **分步执行**：可按“下载数据 → 计算因子 → 训练模型 → 回测对比”的顺序逐段排查。
7. **核对数据质量**：检查 `data/all_etf_factors.csv` 是否存在缺失值或字段缺失。
